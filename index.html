  • Philosophers

    • Philosophers is a project that focuses on solving the classic synchronization problem known as the "Dining Philosophers Problem." The goal is to simulate a scenario where multiple philosophers, sitting at a table, alternately think and eat, but they need to share limited resources (forks) without causing deadlocks or race conditions. The project requires designing a solution using semaphores to synchronize access to shared resources between processes and to avoid issues such as deadlock, starvation, and data corruption. Instead of using threads, we implemented the solution using multiple processes, adding complexity in managing inter-process communication and synchronization. This project provides a deep understanding of concurrency control, process management, and the implementation of efficient, process-safe systems, with a focus on utilizing semaphores to ensure proper synchronization between processes.

  • Minishell

    • Minishell is a project from the 42 school that involves creating a simple Unix shell program.
    The goal is to design and implement a basic command-line interface (CLI) that can execute commands, handle pipes, input/output redirection, and manage environment variables, all while mimicking the behavior of a real shell. The project focuses on handling the parsing of user input, managing processes using fork and exec system calls, and implementing built-in commands such as cd, echo, exit, pwd, env, export and unset.
    Additionally, the shell must properly manage foreground and background processes, handle signals (such as SIGINT and SIGQUIT), and support process termination.
    One of the key challenges is to correctly handle command-line syntax, including parsing for operators like pipes and redirections. The project also requires handling errors and edge cases in a way that ensures the shell runs smoothly.
    The goal is to understand the inner workings of shells, improve skills in process management, system calls, signal handling, and memory management, while also developing a deeper knowledge of how operating systems execute commands in a shell environment.

  • Cub3D

    • Cub3D is a project from the 42 school that involves creating a 3D raycasting engine, simulating a first-person perspective similar to classic 3D games like Wolfenstein 3D. The goal is to design and implement a graphical engine that can render a 3D environment based on a simple 2D map, using raycasting to project walls, floors, and ceilings onto the screen. The project requires handling basic graphics and input management, implementing the raycasting algorithm to detect intersections with walls, and drawing textures to create a visually rich environment. This includes properly mapping textures onto surfaces based on the player's viewpoint and ensuring they scale correctly.
    Additionally, the project involves parsing a map file that defines the layout of the environment, including walls, floors, and objects, and converting it into a format that the engine can understand. The engine must interpret and validate the map file, handle potential errors, and ensure a seamless experience. The project also focuses on performance optimization to ensure smooth rendering and responsiveness.
    By working on Cub3D, you'll develop a deep understanding of computer graphics, 3D rendering, raycasting algorithms, user input handling, texture mapping, and file parsing, all of which are essential skills in building game engines and graphical applications.

  • CPP_MODULES_42

    • The C++42 program consists of 10 modules, numbered from Module 00 to Module 09, each designed to teach different aspects of the C++ language. These modules progressively introduce both fundamental and advanced topics, providing a comprehensive understanding of C++ programming. Each module contains a series of exercises that focus on specific concepts, allowing you to practice and reinforce your skills.

    MD_00 : introduces the basics of C++, focusing on the differences between C and C++ and providing an introduction to Object-Oriented Programming (OOP).

    MD_01 : covers memory allocation, references, and pointers to members, teaching you how to manage memory efficiently and work with C++-specific features.

    MD_02 : dives into Ad-hoc polymorphism, overloading functions, and the orthodox canonical class pattern in C++, providing deeper insights into object-oriented design.

    MD_03 : focuses on Inheritance, teaching you how to create hierarchical relationships between classes in C++.

    MD_04 : covers subtype polymorphism, including abstract classes and interfaces, enabling you to design flexible and reusable code.

    MD_05 : explores exceptions and error handling using try/catch blocks in C++, helping you manage exceptional conditions in your programs.

    MD_06 : teaches you about type casting in C++, explaining the various types of casts and their appropriate use cases.

    MD_07 : introduces templates, allowing you to write generic functions and classes that can work with different data types.

    MD_08 : focuses on templated containers, iterators, and algorithms, teaching you how to create and manipulate collections of data efficiently.

    MD_09 : explores the various standard containers in C++, helping you understand how to use them effectively for data storage and manipulation.

    Each module is structured to build on previous knowledge, making it easier to grasp advanced C++ concepts while reinforcing the basics.
